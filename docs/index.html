hotc-il2cpp


hotc-il2cpp 是一个特性完整、零成本、高性能、低内存的 Unity 全平台原生 C# 热更新解决方案。

hotc-il2cpp 扩充了 il2cpp 运行时代码，使它由纯 AOT runtime 变成 AOT + Interpreter 混合 runtime，进而原生支持动态加载 assembly，从底层彻底支持了热更新。使用 hotc-il2cpp 技术的游戏不仅能在 Android 平台，也能在 iOS、WebGL 等所有 il2cpp 支持的平台上高效运行。

由于 hotc-il2cpp 对 ECMA-335 规范的良好支持以及对 Unity 开发工作流的高度兼容，Unity 项目在接入 hotc-il2cpp 后，可以几乎无缝地获得 C# 代码热更新的能力，开发者不需要改变日常开发习惯和要求。

欢迎拥抱现代原生 C# 热更新技术！

hotc-il2cpp 首次实现了将 Unity 平台的全平台代码热更新方案的工程难度降到几乎为零的水平。

特性

完整 ECMA-335 支持

近乎完整实现了 ECMA-335 规范，支持动态加载 assembly。

零学习成本

对绝大多数开发者来说写代码近乎没有限制。热更新代码与 AOT 代码无缝工作，可以随意写继承、泛型、反射之类的代码。

完全多线程支持

完全支持多线程，包含但不限于 volatile、ThreadStatic、async Task 等相关功能和特性。

极致性能

实现了极其高效的解释器，所有指标都大幅优于其他热更新方案。支持 Token Threaded 调度，性能提升 20-30%。

低内存占用

热更新脚本中定义的类跟普通 C# 类占用一样的内存空间，远优于其他热更新方案。

全平台支持

支持所有 il2cpp 支持的平台：Android、iOS、WebGL、Windows、macOS、Linux。

支持的版本与平台

  • 支持 2019.4.x、2020.3.x、2021.3.x、2022.3.x、2023.2.x、6000.x.y 全系列 LTS 版本
  • 支持所有 il2cpp 支持的平台
  • AndroidiOSWebGLWindowsmacOSLinux

工作原理

hotc-il2cpp 从 mono 的 mixed mode execution 技术中得到启发，为 Unity 的 il2cpp 之类的 AOT runtime 额外提供了 interpreter 模块，将它们由纯 AOT 运行时改造为 "AOT + Interpreter" 混合运行方式。

│  │  Interpreter  │  │  Threaded    │            │

│         └────────┬────────┘                     │
│  ┌───────────────┴───────────────┐             │
│  │     IL Parser & Type System   │  ← 共享核心  │
│  └───────────────┬───────────────┘             │
│  ┌───────────────┴───────────────┐             │
│  │     Unity Bridge (C API)      │  ← CoreModule│
│  └───────────────────────────────┘             │
└─────────────────────────────────────────────────┘

更具体地说，hotc-il2cpp 做了以下几点工作：

  • 实现了一个高效的元数据 (dll) 解析库
  • 改造了元数据管理模块，实现了元数据的动态注册
  • 实现了两种解释器模式：经典模式和 Token Threaded 模式
  • 实现了高效的线程本地执行上下文，缓存行对齐
  • 实现了 Arena 分配器，零碎片内存管理
  • 实现了 JIT 编译器，支持热点方法原生代码生成
  • 额外提供大量的 instinct 函数，提升解释器性能
